## はじめに：「何を予測するか」を決める前の確認

機械学習の文脈では、精度を上げるための努力の多くはアルゴリズムの選定やハイパーパラメータ調整に向かいがちだ。しかし材料・化学データの実務では、しばしば異なる場所にボトルネックがある。

**目的変数の設計**だ。

「引張強度を予測する」「腐食速度を予測する」は、一見シンプルな問いに見える。しかし次の判断を誤ると、いくらモデルを工夫しても精度が出ない、あるいは正しく出た精度を正しく使えない状況になる。

- 目的変数の分布形状は正規分布に近いか。右歪みなら対数変換が必要かもしれない
- 対数空間で学習したモデルの予測値を元のスケールに戻すとき、単純に exp() をかけていないか
- 複数の特性を 1 つの「総合スコア」に合成して目的変数にしていないか

本記事では、これら 3 つのポイントを Fe/Ni/Cr 合金の引張強度データと腐食速度データを使って実証する。デモ設定は前記事（[SHAP 解析と逆解析](https://zenn.dev/efta1989/articles/shap-inverse-analysis-for-materials)）と同じ Fe/Ni/Cr 合金系だが、本記事はサンプル数・目的変数が異なる別データであり、「モデルを作る前の設計」の話として位置づけている。

記事内の Python コードは説明用の断片です。デモデータ生成を含む実行可能版は GitHub の [code.ipynb](https://github.com/mf-m-f/zenn-content/blob/main/notebooks/target-variable-design/code.ipynb) を参照。（[Colab で開く](https://colab.research.google.com/github/mf-m-f/zenn-content/blob/main/notebooks/target-variable-design/code.ipynb)）

## 1. 目的変数の分布を確認する

### なぜ分布形状を確認するのか

多くの回帰モデル（Ridge、GPR など）は、内部的に「残差が正規分布に従う」という仮定を持っている。目的変数そのものが大きく右歪みしているとき、この仮定が崩れる。損失関数（MSE など）は絶対値の大きなサンプルに支配され、小さな値のサンプルの予測精度が極端に低下する。

まず確認すべきは**ヒストグラム**と **QQ プロット**だ。

**QQ プロット（Quantile-Quantile プロット）** とは、データの分位数と正規分布の理論的な分位数を突き合わせた図だ。横軸に「正規分布なら何番目の分位点がこの値になるか」、縦軸に「実データの値」をプロットし、両者が一致していれば点が一直線に並ぶ。右端の点が直線より上に外れていれば「実データは理論値より大きな外れ値を持つ（右歪み）」、全体が S 字状に曲がっていれば「裾が重い分布」を示す。

直線からの外れ具合を相関係数 r で数値化できるが、**r だけで変換の要否を決めてはいけない**。r は右歪み・左歪みのどちらでも下がりうるし、中央の波打ち（完全な正規ではないが大きな歪みもない）を見逃しやすい。**どの裾が外れているか**を図で確認してから変換を選ぶ。

```python
import scipy.stats as stats
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(10, 4))

axes[0].hist(tensile_strength, bins=12, edgecolor='black', alpha=0.7)
axes[0].set_xlabel('引張強度 [MPa]')

(osm, osr), (slope, intercept, r_ts) = stats.probplot(tensile_strength, dist='norm')
axes[1].plot(osm, osr, 'o', alpha=0.7)
axes[1].plot(osm, slope * np.array(osm) + intercept, 'r-')
axes[1].set_title(f'QQ プロット（r = {r_ts:.3f}）')
```

### デモデータでの結果（引張強度）

`code.ipynb` の Section 1 では、引張強度（500〜700 MPa 程度）で QQ 相関係数 **r = 0.983** となった。0.98 以上なので「ほぼ正規」と言えるが、プロット中央に軽い波打ちがあり、完全な正規分布ではない。**右歪みは弱く、対数変換は不要**という判断になる。

### 正規性の判断基準

| 指標 | 正規分布に近い | 要変換 |
|:---|:---|:---|
| ヒストグラム形状 | 左右対称に山が 1 つ | 右に裾が長い（右歪み） |
| QQ プロット相関係数 r | 0.98 前後（形も確認） | 0.95 未満、または右裾が明確に外れる |
| 最大値 / 最小値の比 | 5 倍以内 | 10 倍超（桁違い） |

腐食速度・反応速度定数・欠陥密度のように数桁にわたって変化する変数は、ほぼ例外なく対数変換が必要になる。

ここで重要な前提を整理しておく。**対数変換が有効なのは右歪み（右に裾が長い）の場合だけだ。** 左歪み（左に裾が長い）には対数変換は効かない。

| 歪みの方向 | 典型的な物性値 | 有効な変換 |
|:---|:---|:---|
| **右歪み**（大きな値が外れる） | 腐食速度、反応速度定数、欠陥密度 | 対数変換、Box-Cox（λ < 1） |
| **左歪み**（小さな値が外れる） | 変換効率・純度など上限に張り付く指標 | 反転（max − y）後に対数変換、Yeo-Johnson 変換 |

材料・化学データでは右歪みの方が圧倒的に多い。本記事の Section 2 以降は右歪みの対処法に絞って説明する。

## 2. 対数変換が有効なケース

### 右歪みになる物性値の特徴

腐食速度（μm/year）を例にとると、Cr が 10〜20% の Fe/Ni/Cr 合金で pH と温度を変えた実験では、腐食速度が数十倍の範囲に分布することがある。デモデータでは **1.3〜84.8 μm/year（レンジ比 63 倍）** となった。

```python
# 腐食速度は対数正規分布に従いやすい（code.ipynb Section 2）
log_cr = (
    2.0 - 0.15 * Cr_c + 0.05 * Ni_c
    + 0.5 * (7 - pH) + 0.02 * T_env
    + np.random.normal(0, 0.4, m)      # m = 50
)
corrosion_rate = np.exp(log_cr)   # μm/year
```

QQ 相関係数は **変換前 0.856 → 対数変換後 0.990** と大きく改善した。

Ridge 回帰で LOOCV R² を比較すると次のとおりだ。

| モデルの学習スケール | LOOCV R² |
|:---|:---|
| 元スケール（μm/year） | **0.424** |
| 対数スケール（log）→ 単純 exp で逆変換して評価 | **0.513** |

差は大きくないが、対数変換で R² が改善する。ノイズ水準や合金系によってこの差は変わるため、実際のデータでは必ず両方試して比較することを勧める。

### 変換関数の使い分け

| 関数 | 適用条件 | 注意点 |
|:---|:---|:---|
| `np.log(y)` | y > 0 が保証されている | ゼロ値があるとエラー |
| `np.log1p(y)` | y ≥ 0 の場合（ゼロを含む） | 微小な y のときに精度が高い |
| `scipy.stats.boxcox(y)` | y > 0、最適な λ を自動推定 | λ の保存と逆変換 `inv_boxcox` が必要 |
| `sklearn.preprocessing.PowerTransformer(method='yeo-johnson')` | 負の値も含む | Pipeline に組み込めるため管理が楽 |

材料データで最もよく使うのは `np.log` か `np.log1p` だ。

## 3. 逆変換バイアスと Duan 補正

### 対数スケールで学習した後の落とし穴

対数変換を使うと精度は上がる。しかしここに見落とされやすい問題がある。

**対数空間での予測値 $\hat{z}$ を元のスケールに戻す際、単純に $\exp(\hat{z})$ をかけると系統的な過小予測が生じる。**

これは数学的な必然で、モデルの欠陥ではない。

### Jensen の不等式とバイアスの発生

対数空間での目的変数を $z = \log y$ とする。残差 $\varepsilon$ が存在し、$z = \hat{z} + \varepsilon$ となる。

$\exp$ 関数は上に凸なため、Jensen の不等式から

$$E[\exp(\hat{z} + \varepsilon)] = \exp(\hat{z}) \cdot E[\exp(\varepsilon)]$$

が成り立つ。残差が $\varepsilon \sim \mathcal{N}(0, \sigma^2)$ に従うとき、$E[\exp(\varepsilon)] = \exp(\sigma^2/2) > 1$。

つまり、$\exp(\hat{z})$ は $E[y]$ を $\exp(\sigma^2/2)$ だけ**系統的に過小評価**する。

### Duan の smearing 補正

Duan（1983）が提案した補正は、残差分布を考慮して逆変換バイアスを除去する。

$$\hat{y}_{\text{corrected}} = \exp\!\left(\hat{z} + \frac{\hat{\sigma}^2}{2}\right)$$

$\hat{\sigma}^2$ は対数空間での訓練残差の分散だ。`code.ipynb` の LOOCV 実装は次のとおり。

```python
log_preds_loo = np.zeros(m)
sigma2_per_fold = np.zeros(m)

for train_idx, test_idx in LeaveOneOut().split(X_cs):
    X_tr, X_te = X_cs[train_idx], X_cs[test_idx]
    y_tr = log_cr[train_idx]
    mdl  = Ridge(alpha=1.0).fit(X_tr, y_tr)
    log_preds_loo[test_idx] = mdl.predict(X_te)
    res = y_tr - mdl.predict(X_tr)
    sigma2_per_fold[test_idx] = res.var()

sigma2_est = sigma2_per_fold.mean()
y_naive = np.exp(log_preds_loo)
y_duan  = np.exp(log_preds_loo + sigma2_est / 2)
```

### 効果の確認——指標の選び方が重要

Jensen バイアスは **平均レベル** `mean(予測) / mean(実測)` で確認する。1 未満なら系統的過小、1 に近いほどバイアスが小さい。

一方、サンプルごとの `mean(予測 / 実測)` は実測が小さい点ほど膨らみやすく、**補正の効果を誤って評価しがち**だ。Duan 補正後にこの指標が 1 から離れるのは正常で、補正が「逆効果」に見えることがある。

`code.ipynb` Section 3 の実行結果：

| 指標 | 補正なし（単純 exp） | Duan 補正後 |
|:---|:---|:---|
| **主指標** `mean(予測) / mean(実測)` | **0.937**（系統的過小） | **1.011**（ほぼバイアスなし） |
| 参考 `mean(予測/実測)` | 1.100 | 1.187 |

推定 $\hat{\sigma}^2 = 0.152$、補正係数 $\exp(\sigma^2/2) \approx 1.079$。全サンプルの予測水準が平均で約 8% 過小評価されていたのが、補正でほぼ解消される。

散布図では左右で点の並びはほぼ同じ（Duan 補正は各点を同じ比率で持ち上げるだけ）。バイアス改善は数値で確認する。

### 補正が必要なケースと不要なケース

| 用途 | 補正の要否 | 理由 |
|:---|:---|:---|
| **予測値の絶対値を報告する**（「この配合なら腐食速度は〇〇 μm/year」など） | **必要** | バイアスにより系統的に過小評価されるため |
| **精度指標（R²、MAE）を元スケールで計算する** | **必要** | バイアスがスコアに影響するため |
| **逆解析・最適化**（予測値が最大になる組成・条件を探す） | **不要** | exp() は単調増加なので、対数スケールでの大小順位 ＝ 元スケールでの大小順位 |
| **次の実験候補を選ぶ**（上位 N 件の候補を絞る） | **不要** | ランキング目的なら対数スケールのまま比較して問題ない |

「どの条件が一番良いか」を比較するだけなら対数スケールで完結する。「どのくらい良いか」という絶対的な値が必要になったとき、はじめて補正が必要になる。

### 対数スケールで逆解析するときの具体的な手順

対数スケールで学習したモデルを使って逆解析する場合、**目標値も対数変換してから最適化する**。一貫して対数空間の中で完結するため、逆変換バイアスを気にする必要はない。

```python
target_log = np.log(target_original)

def objective(x):
    pred_log = model.predict(scaler.transform([x]))[0]
    return (pred_log - target_log) ** 2

result = minimize(objective, x0=x_init, method='SLSQP', constraints=constraints)
```

最適条件を実験に使うだけなら逆変換は不要だ。予測腐食速度を μm/year で報告したいときだけ Duan 補正を加える。

```python
pred_log = model.predict(scaler.transform([optimal_conditions]))[0]
pred_original = np.exp(pred_log + sigma2_est / 2)
```

## 4. 目的変数の「圧縮」問題

### なぜ複数特性を 1 つのスコアに合成したくなるか

実際の材料開発では、1 つの特性だけを最大化すれば良いことはほとんどない。たとえば高強度鋼の開発では「引張強度」と「硬さ（耐摩耗性）」の両方を高くしたい。このとき、2 つの目的変数を個別に扱うより「総合スコア = 標準化引張強度 + 標準化硬さ」のような合成指標を 1 つ作り、それを目的変数にする誘惑がある。

### なぜ圧縮スコアではうまくいかないか

個々の物性値（引張強度、硬さ）は、組成・プロセス条件との間に比較的シンプルな関係を持っている。一方、標準化して足し合わせた合成スコアへのマッピングは、それぞれの物性の非線形な相互作用を含み、**最も予測しやすい単一物性より精度が出にくくなる**ことがある。

デモでは硬さを `0.28 × 引張強度 + ノイズ` として生成し、圧縮スコアは両者を標準化して合算した。

```python
for name, y in [
    ('引張強度 [MPa]',      tensile_strength),
    ('硬さ [HV]',           hardness),
    ('圧縮スコア（TS+HV）', composite_score),
]:
    y_pred = cross_val_predict(Ridge(alpha=1.0), X_alloy_s, y, cv=loo)
    print(f"{name:26s}  LOOCV R² = {r2_score(y, y_pred):.3f}")
```

`code.ipynb` Section 4 の実行結果：

```
引張強度 [MPa]                  LOOCV R² = 0.626
硬さ [HV]                     LOOCV R² = 0.187
圧縮スコア（TS+HV）                LOOCV R² = 0.466
```

引張強度単体（0.626）に比べ、圧縮スコア（0.466）は明らかに低下している。硬さ単体（0.187）よりは高いが、それは硬さが引張強度と強く相関しているためで、「合成したから予測しやすくなった」わけではない。**最も予測しやすい物性の構造を壊して中間の精度に落ち、しかもスコアの物理単位・解釈が不明確になる**——これが圧縮の落とし穴だ。

### 推奨する設計

| アプローチ | 評価 |
|:---|:---|
| 合成スコアを 1 つ作ってそれを予測 | 単一物性より精度が出にくいことが多い。解釈が困難 |
| 各物性値を個別に予測 → 後処理で統合 | 精度が出やすく、各物性値の解釈も明確に保てる |

複数の特性を同時に最適化したい場合は、各モデルの出力を使ってパレート最適化（Pareto Front）を行うのが正しい設計だ。

## 5. 測定誤差をモデルに組み込む（発展的な話題）

※ 本節は `code.ipynb` ではデモしていない概念整理である。

標準的な回帰モデルは、すべてのデータポイントを等しく信頼して損失関数を最小化する。しかし実験データには測定誤差が含まれており、その大きさはサンプルによって異なることが多い。

測定ごとの分散 $\sigma_i^2$ が既知（あるいは反復実験から推定可能）な場合、**Heteroscedastic Gaussian Process Regression（不均一分散 GPR）** の利用を検討する価値がある。標準 GPR が全領域で一定のノイズ $\sigma^2$ を仮定するのに対し、Heteroscedastic GPR ではノイズの分散を各サンプルの測定誤差から直接設定できる。

⚠️ **ただし各サンプルの測定誤差 $\sigma_i^2$ の推定値が必要だ**。同一条件での反復測定値がない場合（材料実験では珍しくない）、この手法は適用できない。測定誤差の見積もりがなければ、通常の GPR か Ridge 回帰を使い、不確実性は予測の信頼区間で管理するのが現実的だ。

## 6. 設計判断のフロー

```
目的変数を決める
   ↓
① 分布確認（ヒストグラム / QQ プロット）
   ├── 右歪みが弱く r ≳ 0.98 → 対数変換不要、そのまま学習
   └── 右歪みが強い / r < 0.95 → 対数変換（log / log1p / Box-Cox）
          ↓
       対数空間で学習
          ↓
② 絶対値や元スケールの R² を報告する？
   ├── はい → Duan 補正 exp(ẑ + σ²/2)（効果は mean(予測)/mean(実測) で確認）
   └── いいえ（ランキング・逆解析のみ）→ 対数スケールのまま
   ↓
③ 目的変数は 1 つにするか複数にするか
   ├── 単一物性 → そのまま
   └── 複数物性 → 個別に予測し、後処理でパレート最適化
```

## おわりに

目的変数の設計は、アルゴリズムの選定より地味だが、精度の上限を決める。

本記事のポイントを 3 行でまとめると：

- **分布を確認してから学習する**──QQ プロットは形を見る。右歪みなら対数変換
- **対数空間で学習したら、絶対値報告時は Duan 補正で戻す**──効果は `mean(予測)/mean(実測)` で確認（`mean(予測/実測)` は誤解を招きやすい）
- **目的変数を圧縮しない**──複数特性は個別に予測し、後処理で統合

次の記事では、「何を入力にするか」の側、すなわち特徴量の設計・選択と Pipeline によるデータリーク防止を扱う。
